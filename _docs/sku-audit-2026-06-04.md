# SKU audit — BOM emissions vs catalogue (2026-06-04)

Cross-referenced every SKU the 12 wizard styles emit (`server/services/bom-calculator.ts`)
against the calc `products` catalogue (597 codes, Neon). Classified each family. Goal:
know what's price-ready, what needs seeding, and what the solver emits as a placeholder
despite a real SKU existing.

## Class A — Confirmed mapped (PRICE-READY) — the 5 aluminium styles
Every hardware family for alu-pool (BARR/Blade/Tubular) + alu-bal (BARR/Blade) exists in the
catalogue with finish variants:
`AR-5800-FP·AR-1050-FPBP·AR-1500-FMID·AR-1500-FMLR·XP-TP·XP-DR·XP-DC-2P·BR-PANEL-1733·BR-PANEL-2205·BR-BR60·BR-BR25·BR-BRCAP·BR-1280-BP·BR-DC-2P·BR-1800·BR-DR·BR-GATE·BLA-PNL-1700·BLA-PNL-2200·BLA-GATE·FF-BH-OPEN·SS-1300-BP·SS-DC·SS-1800·SS-BH4·SS-BSWIV·SS-FTG·SS-FTP·GROUT-SETFAST·S-110LAG·S-120ROD·SOUD-CA1400·GS160LAG·GS150ROD·GS-DN·CSK-100-4PK·ML-TL·TC-H-AT`.
→ The 5 aluminium styles are essentially SKU-complete (pending price import).

## Class B — Glass panels: family exists, but MADE-TO-SIZE + placeholder mismatches
- **Bal glass** families exist at 50mm steps: `1000FBG` (23, 15mm/1000H), `970NTG` (25, 12mm/970H),
  `1280S` (14, standoff 1280H). But the solver emits the **exact equalised width** (e.g.
  `1000FBG-1187`), which is NOT a catalogue row. → needs a made-to-size pricing rule
  (nearest-stock-size, or price per m²).
- **Pool glass**: real family is `12N-{w}` (64 codes) — but the solver emits placeholder
  `GP-{w}-1200-12`. → rewire to `12N-`.
- Gate/raked/custom/hinge glass: `GP-GATE-`/`RP-L`/`RP-R`/`GP-CUSTOM-`/`GP-HINGE-` placeholders →
  map to real pool-glass variants (soft-close gate glass is `12NPH-`/`12PGG-`).

## Class C — UNSEEDED — must be created in the catalogue (the real gaps)
- **★ Entire VersaTilt channel family — NONE exist** (both 15mm and HD). Blocks pricing for
  `glass-bal-channel` AND `glass-bal-channel-hd`:
  `VER-4200-DMK-{B/SA}` · `VER-HD-3600-DMK-{B/SA}` · `VER-PPKIT-15MM` · `VER-HD-PPKIT-17-4PK` ·
  `VER-WASHER-14PK` · `VER-HD-WASHER-18PK` · `VER-15KIT-RUB` · `VER-HD-17KIT-RUB` ·
  `VER-2DMEP-{finish}` · `VER-HD-2DMEP-{finish}` · `VER-PINS`.
- **HD glass** `1000SGP1752-{w}` (17.52 SGP) + the **17.52 rail runner** (`SER35-RUB` only stocks
  `-12MM`/`-15MM`, no 17.52). Both derived.
- **Glass laminated `-LAM`** lines (AS1288 ≥5m for spigots/standoffs/channel) — zero `-LAM`
  codes in the catalogue.
- **Pool channel legacy** `VC-4200-*` / `CFC-300` / `CEC-STD` — made-up SKUs the
  `glass-pool-channel` path emits; should map onto the real VersaTilt family once seeded.

## Class D — Solver emits PLACEHOLDER but the real SKU EXISTS → rewire solver (code, no seeding)
- **★ Rail** — solver emits `RAIL-{type}-5800-{material}-{finish}` + `RAIL-{type}-{term}-…`, but the
  real **35-Series family is fully catalogued**: `SER35-R5800-{B/MW/SA}` (rail) · `SER35-J-SA`
  (inline joiner) · `SER35-J90-SA` (90°) · `SER35-VJA-SA` (adj. corner) · `SER35-WB-`/`WBEXT-`
  (wall plates) · `SER35-EC-` (end caps) · `SER35-RUB-15MM` (glazing). Affects EVERY balustrade
  style (all use the 35-Series rail). High-value rewire.
- **Standoff hardware** — emits `STANDOFF-50-{finish}`; real bodies are `GSA-{depth}-{finish}` (8)
  + `GS50{depth}{finish}` (7). Rewire `glass-bal-standoffs`.
- **Pool gate latch** — `PAP-POLARIS` not found (soft-close). `ML-TL-TC-H-AT` (Magna+TruClose)
  exists. Verify the real soft-close hardware SKU (`12NPH-` soft-close glass exists).

## Recommended order
1. **Seed the VersaTilt channel family** (15mm + HD) — unblocks the 2 channel styles. ~13 stems × finishes.
2. **Rewire the rail solver** `RAIL-` → `SER35-*` — correct rail pricing across all balustrade styles (code only).
3. **Rewire** standoff `STANDOFF-50-` → `GSA-/GS50`, and pool glass `GP-` → `12N-` (code only).
4. **Seed** HD glass `1000SGP1752`, the glass `-LAM` laminated lines, and the 17.52 rail runner.
5. **Verify** the pool soft-close gate latch (`PAP-POLARIS`).
6. Then the price import (`bh_storefront.retail_price_inc_gst`) attaches to a clean SKU set.
