Goal:
Eliminate cases where a required gate disappears and totals no longer equal section length. Add a hard invariant, fix gap counting, and reproduce the exact 5000 mm scenario from the screenshot in a test.

Bugs to address:
- Gate Required = true but no 'gate' segment gets inserted (composer continues with only panels + between gaps).
- Length conservation broken when gate is near/right edge or after flips/toggles.
- Equalize sometimes runs BEFORE fixed elements (hinge/gate/gaps) are placed.

Tasks:

1) Add a strict invariant in the resolver/composer
   File: server/services/resolve.ts (or your composition module)
   - After assembling segments but BEFORE finalizing:
     * If selection.gate_required === true:
         - Assert exactly ONE segment with kind === 'gate'.
         - Assert mount_mode === 'GLASS_TO_GLASS' ⇒ NO 'post-anchor' is fine; POST/WALL ⇒ must have a 'post-anchor' segment on hingeSide.
     * Compute sumMm = startGap + Σ(panel widths) + Σ(between gaps actually used) + (hingeGap + latchGap around the gate) + endGap [+ hinge panel width if G2G].
       - Replace the betweenGap at gate interfaces with hingeGap/latchGap (no double-counting).
       - If |sumMm - runLengthMm| > 2, throw a 400 with { code:"LENGTH_INVARIANT", expected: runLengthMm, got: sumMm, segments } and DO NOT fall back to a 2×2000 layout.
   - Add a trace entry 'invariant-check' with { sumMm, runLengthMm, ok: true/false }.

2) Fix ordering: equalize AFTER fixed elements
   - Ensure the sequence is built in this order:
     a) startGap
     b) left-side std panels (if any)
     c) (GLASS_TO_GLASS) hinge panel width (explicit) + hingeGap
        (POST/WALL) post-anchor + hingeGap
     d) gate (gateWidth)
     e) latchGap
     f) fill remainder via equalizePanels(...) into N panels using step=50
     g) endGap
   - Equalize runs ONLY on the computed remainder span(s).

3) Gap math helper
   File: shared/calc/gaps.ts (new)
   export function computeBetweenGapCount(sequence: Array<'panel'|'hinge'|'gate'|'post'|'gap'>): number
   - Between gaps appear between adjacent **glass** elements (panel or hinge panel), NOT between:
       • start/end gaps
       • hingeGap/latchGap around the gate
       • posts
   - Use this to build the final sum precisely and avoid double-counting.

       • exactly one 'gate' segment present
