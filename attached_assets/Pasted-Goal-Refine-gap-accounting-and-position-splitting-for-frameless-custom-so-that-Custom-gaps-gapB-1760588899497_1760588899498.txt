Goal:
Refine gap accounting and position splitting for frameless_custom so that:
- Custom gaps (gapBeforeMm/gapAfterMm) are honored and replace connector gaps at those interfaces.
- Connector (between) gaps auto-insert only between adjacent variable panels.
- RIGHT position places the custom panel within ≤ 200 mm of the intended anchor after snapping to the 50 mm grid.
- Length is conserved (±2 mm tolerance for non-gate).
- All frameless_custom tests pass.

Key Rules (make these the single source of truth)
1) Gap precedence:
   a) If a custom gap (before/after) is specified next to the custom panel, it REPLACES a connector gap at that interface.
   b) Connector (between) gaps appear ONLY between two variable glass panels (panel | panel).
   c) No connector gaps adjacent to start or end gaps.
   d) No double counting: a boundary can contribute at most one gap (custom OR connector).
2) Position splitting (LEFT=0.0, MIDDLE=0.5, RIGHT=1.0):
   a) Compute the target anchor along the variable-panels block (sum of equalized widths + connector gaps).
   b) Snap the split boundary to the nearest BETWEEN boundary (i.e., between panels), not through a panel.
   c) For RIGHT=1.0, clamp to the final between boundary if any; if none (N=1), split to the right of the single panel (i.e., left set empty).
   d) Distance from requested split anchor to actual chosen between boundary must be ≤ 200 mm after snapping; otherwise log a trace warning but proceed deterministically.
3) Single custom panel edge case:
   a) If N=0 (no variable panels), the layout is [startGap] + [custom_gap_before?] + [custom_panel] + [custom_gap_after?] + [endGap], and still must conserve length.
   b) If N=1, there is no connector gap around the single variable panel; only custom gaps may appear if specified.
4) Length conservation invariant:
   After composing segments, sum widths and assert |sum − runLength| ≤ 2 mm (non-gate). On fail, throw LENGTH_INVARIANT with segments dump.
