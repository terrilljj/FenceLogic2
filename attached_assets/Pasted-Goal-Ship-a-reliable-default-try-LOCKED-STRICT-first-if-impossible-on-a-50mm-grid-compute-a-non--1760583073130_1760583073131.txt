Goal:
Ship a reliable default: try LOCKED_STRICT first; if impossible on a 50mm grid, compute a non-negative residual end gap and report variance. Keep LOCKED_STRICT behind a flag (STRICT_END_GAP=1) for QA. Add a small Advisor to suggest feasible end gaps near the requested value. Update tests and run QA.

Tasks:

1) Default policy
- In resolver (frameless variant), set default EndGapPolicy = 'LOCKED_OR_RESIDUAL'.
- If process.env.STRICT_END_GAP === '1', force 'LOCKED_STRICT'.
- Trace: always add { source:'end-gap-policy', policy, lockedTried:true/false, residualUsed:true/false, varianceEndGap:mm }.

2) Policy behavior
- Keep current LOCKED_STRICT path (exact requested end gap; if impossible, 400 UNREACHABLE).
- Implement fallback (RESIDUAL) for LOCKED_OR_RESIDUAL:
  a) Recompute panel widths ignoring end gap: targetPanels0 = section - F (no end gap applied).
  b) Find N via findFeasibleN, equalizePanelsExact(targetPanels0, N, step=50, min, max).
  c) Compute residualEndGap = section - (startGap + sum(widths) + (N-1)*betweenGap + hingeGap + gateWidth + latchGap + (hingePanelWidth?))
  d) Ensure residualEndGap >= 0; if negative, try increasing N up to N_max; if still negative → 400 UNREACHABLE with reason 'residual_negative'.
  e) Record varianceEndGap = residualEndGap - requestedEndGap in trace/response warnings.

3) Advisor endpoint (read-only)
- Add file shared/calc/endgapAdvisor.ts with:
  export function nearestFeasibleEndGaps(params): { requestedFeasible:boolean, candidates:Array<{endGapMm:number, deltaMm:number, N:number}> }
  Logic: sweep endGapMm from requested-50 to requested+50 (step 5 or 10), try LOCKED_STRICT solve each; collect up to 3 closest feasible.
- Add GET /api/debug/endgap-advice?variant=<frameless>&run=<mm>&start=<mm>&between=<mm>&end=<mm>&gate=<mm>&hinge=<mm>&mode=<G2G|POST|WALL>
  Return JSON from advisor. Do not write DB.
